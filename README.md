# RV1126B Smart Security Access Control

基于 RV1126B 开发板的智能安防门禁原型系统，集成人脸检测、人脸识别、本地大模型问答、语音输入播报、触摸屏 UI 和低功耗控制。

## 功能

- USB 摄像头实时采集画面，在屏幕左侧显示人脸检测和识别结果。
- 支持已注册人员识别和陌生人统计。
- 陌生人停留超过阈值后自动报警，并通过大模型对话框和语音播报提示。
- 右侧运行本地大模型界面，支持文字输入、语音输入和自动播报。
- 可通过自然语言查询安防状态，例如当前人数、陌生人累计次数、报警次数。
- 可通过自然语言控制报警间隔，最小间隔限制为 3 秒。
- 支持触摸屏操作和系统拼音输入法。
- 支持低功耗模式：屏幕 1 分钟无操作且无活跃任务时关闭大模型和语音模块；点击屏幕或检测到人脸后恢复。

## 亮点

- 边缘端运行：核心识别和交互都部署在 RV1126B 本地，不依赖云端服务。
- 视觉 AI 与大模型融合：大模型不只是聊天，还能查询和控制人脸识别模块。
- 多模态交互：摄像头、触摸屏、文字输入、语音输入和语音播报共同工作。
- 面向安防门禁场景：包含陌生人停留报警、累计统计、报警间隔设置和状态问答。
- 面向低算力设备优化：支持大模型开关、TTS/ASR 状态保护、低功耗进入和唤醒策略。

## 目录

- `face_ai/`：人脸检测、人脸识别、注册和屏幕显示相关 C/C++ 源码及脚本。
- `llm_ui/`：本地大模型 GTK 触摸屏界面、语音交互、低功耗逻辑和部署脚本。
- `docs/`：项目说明文档。

## 不包含的文件

仓库不提交大模型权重、人脸照片、运行日志、编译产物和系统 SDK。需要的模型文件请参考 [MODEL_FILES.md](MODEL_FILES.md)。

## 板端默认路径

项目运行时默认使用以下路径：

- 大模型 UI：`/userdata/llm/ui`
- 大模型文件：`/userdata/llm/rkllm/models`
- 语音模型：`/userdata/llm/voice`
- 人脸识别运行目录：`/userdata/face_ai`
- 人脸库：`/userdata/face_ai/faces`

## 运行方式

在开发板上恢复运行环境后，启动 systemd 服务：

```bash
systemctl enable llm-system-ime.service
systemctl restart llm-system-ime.service
```

也可以通过 `llm_ui/start_board_system_ime.sh` 从电脑端启动板子上的屏幕界面。

## 当前硬件

- 主控：RV1126B 开发板
- 摄像头：USB 摄像头
- 显示：MIPI 触摸屏
- 音频：USB 一体式麦克风/扬声器模块
