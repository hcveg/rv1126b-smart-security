# RV1126B 智能安防门禁项目说明

## 项目定位

本项目使用 RV1126B 作为端侧主控，在 MIPI 屏上同时运行人脸识别和本地大模型对话界面。系统不依赖云端，摄像头画面在板端完成采集、检测、识别、事件统计和报警，大模型界面用于查询安防状态、执行简单控制命令和完成基础问答。

## 当前功能

1. 左侧实时人脸识别
   - USB 摄像头采集画面。
   - RetinaFace NPU 模型做人脸检测。
   - ArcFace NPU 模型提取人脸特征。
   - 支持 hzj、lyc 两个人脸库。
   - 未注册人员显示为 Unknown。

2. 右侧本地大模型对话
   - 使用 RKLLM 后端运行 MiniCPM4-0.5B 量化模型。
   - 支持系统 IBus 拼音输入法和触摸键盘。
   - 支持普通简短问答。

3. 大模型控制人脸模块
   - 输入“打开摄像头”“打开人脸识别”会打开左侧人脸识别窗口。
   - 输入“关闭摄像头”会停止摄像头和人脸识别。
   - 输入“摄像头状态”会查询当前运行状态。

4. 安防事件统计与自动报警
   - 人脸识别进程每秒写出安防状态到 `/tmp/face_ai_security_status.json`。
   - 陌生人连续稳定出现约 1 秒后，记录事件到 `/tmp/face_ai_security_events.jsonl`。
   - 陌生人在画面内连续停留超过 10 秒时，右侧对话框自动追加一次安防警报。
   - 如果同一张脸随后被识别成已注册人员，会清理对应陌生人轨迹，减少识别抖动造成的重复计数。
   - 可询问：
     - “从开机到现在一共出现了几个陌生人”
     - “现在画面里几个人”
     - “报警几次”
     - “陌生人停留多久”

5. 开机默认启动
   - 当前默认启用 `llm-system-ime.service`。
   - 上电后自动进入“大模型 + 人脸识别 + 拼音输入法”的门禁界面。
   - 独立全屏人脸检测服务 `face-ai-drm.service` 保留，但默认禁用。

## 当前屏幕布局

- 左侧：人脸识别画面，约占屏幕左半边。
- 右上：大模型对话记录。
- 右下：系统触摸键盘和拼音候选框。

## 关键路径

电脑备份目录：

```text
/home/enovo/qs/llm_board_runtime
/home/enovo/qs/face_ai_board_runtime
```

板端运行目录：

```text
/userdata/llm
/userdata/face_ai
```

恢复集成界面：

```bash
/home/enovo/qs/llm_board_runtime/restore_to_board.sh
ssh root@192.168.0.232 'systemctl disable --now face-ai-drm.service; systemctl enable --now llm-system-ime.service'
```

查看安防状态：

```bash
ssh root@192.168.0.232 'cat /tmp/face_ai_security_status.json'
ssh root@192.168.0.232 'tail -20 /tmp/face_ai_security_events.jsonl'
```

## 可以继续完善的方向

1. 门禁动作闭环
   - 增加继电器、舵机或 GPIO 控制，实现“识别到 hzj/lyc 后开门”。
   - 增加开门日志，记录开门时间、人员姓名和置信度。

2. 更完整的事件管理
   - 把陌生人事件保存到 `/userdata`，断电后仍可查询历史记录。
   - 增加抓拍图片，例如陌生人停留超过阈值时保存一张 JPEG。

3. 更稳的人脸识别
   - 每个人采集 15 到 20 张现场照片，覆盖正脸、轻微侧脸、不同距离和不同光照。
   - 对小脸、遮挡、侧脸单独降低置信度，减少误识别。
   - 给每个识别结果显示相似度和投票数，便于现场调阈值。

4. 大模型能力增强
   - 增加固定意图解析，支持“今天来了几个人”“最近一次陌生人什么时候出现”等问题。
   - 对安防问题优先走本地结构化数据，不依赖模型自由生成。
   - 普通聊天仍交给本地小模型处理。

5. 工程化展示
   - 增加系统健康状态：CPU、内存、NPU 推理帧率、摄像头状态。
   - 增加一键清空事件、导出日志、重启人脸模块等按钮。
